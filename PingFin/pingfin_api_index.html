Bank KBC API Documentation

Deze pagina documenteert de API van onze PingFin regular bank. De bank kan werken als OB (Originating Bank) en als BB (Beneficiary Bank).

De GUI gebruikt deze endpoints om payment orders te genereren, verwerken, loggen en controleren. De database draait in MySQL via Docker en wordt gevuld via init.sql.

Use cases

UC1 — OB validation fails

PO_NEW wordt FAILED, geen PO_OUT, fout wordt gelogd.

UC2 — Internal payment

OA en BA zitten bij Bank KBC. ACCOUNTS en TRANSACTIONS worden aangepast.

UC3 — CB validation fails

Externe PO gaat naar PO_OUT, daarna negatieve ACK_IN door CB-fout.

UC4 — BB validation fails

Inkomende PO faalt bij BA-validatie en maakt ACK_OUT FAILED.

UC5 — All validations pass

Succesvolle externe betaling met ACK success en correcte transaction records.
